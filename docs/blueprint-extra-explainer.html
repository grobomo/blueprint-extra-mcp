Blueprint Extra MCP

Browser automation for AI assistants -- using your real browser, real profile, real sessions

[!] The Problem

Without Blueprint

  • Playwright/Puppeteer launch headless browsers
  • Must re-authenticate every session
  • Bot fingerprints get blocked by sites
  • No access to your extensions or cookies
  • Iframe content invisible to AI snapshots
  • One AI session per browser instance

With Blueprint Extra

  • Uses your real Chrome/Firefox/Edge browser
  • Already logged in everywhere
  • Real fingerprint -- not flagged as bot
  • Your extensions and cookies work
  • Same-origin iframe content in snapshots
  • Multiple AI sessions share one browser

[#] Key Files

run-server.js

Entry point. Launches MCP stdio server or relay client based on port availability.

server/src/unifiedBackend.js

All 20+ browser tool handlers. This is where fixes go. Iframe search, click fallbacks.

server/src/statefulBackend.js

Connection state machine. Manages passive/active/connected transitions.

server/src/extensionServer.js

WebSocket server on port 5555. Multiplexes relay clients to single extension.

extensions/manifest.json

Chrome MV3 manifest. Points to chrome/src/background-module.js service worker.

rules/*.md

Claude Code automation rules. Best practices, health checks, V1 page recipes.

[>] Key Commands
enable(client_id="my-project")

Activate browser control. Required before any browser_* tool.

browser_tabs(action="list")

List open tabs. Also: new, attach, close.

browser_navigate(url="...")

Navigate to URL in attached tab.

browser_snapshot()

Get accessible page content including iframe content.

browser_interact(action="click", ref="E42")

Click, type, select, hover, scroll on elements.

browser_evaluate(expression="...")

Execute JavaScript in page context.

browser_take_screenshot()

Capture visual screenshot of current tab.

browser_lookup(selector=".class")

Find elements by CSS selector. Auto-searches iframes.
